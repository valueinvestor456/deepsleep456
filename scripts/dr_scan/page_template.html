DR Fair Price Monitor — เทียบราคาจริง vs Fair Price (Arbitrage)

Fair Price = ราคาหุ้นแม่ × อัตราแลกเปลี่ยน ÷ อัตราแปลงสภาพ (ratio) — ดึงราคาจริง+ratio สดจาก SET, ราคาหุ้นแม่/FX จาก Yahoo Finance

อัปเดต:

⚠️
ข้อมูลเป็น snapshot ไม่ใช่ real-time
ราคาและ ratio ดึงจาก SET (ราคาปิดล่าสุด) + ราคาหุ้นแม่/FX จาก Yahoo Finance ณ เวลาที่ scan — ต้องเทียบกับราคาจริงในกระดานอีกครั้งก่อนตัดสินใจซื้อขายจริงเสมอ ครอบคลุม DR ทั้งหมด 463 ตัวที่มีบน SET (รายชื่อทั้งหมดดึงจาก filter dropdown ของหน้า marketdata เอง) — resolve mapping หุ้นแม่ได้ครบทุกตัวแล้ว ระหว่างเวลาทำการ SET (09:58-16:30 น.) หน้านี้ auto-refresh ทุก ~30 วิ ผ่าน GitHub Actions นอกเวลาทำการเป็น snapshot ล่าสุดของรอบก่อนตลาดปิด

🛑 STOP — จุดตรวจสอบก่อนเชื่อ
  • Ratio ทุกตัวดึงสดจาก SET (`exerciseRatio` ในหน้า factsheet ของแต่ละ DR) ไม่เคยใช้ค่าที่เก็บไว้เก่าเลย — ระหว่างพัฒนาเวอร์ชันแรก (39 DR) พบว่าไฟล์เก่าของโปรเจกต์ (raw/dr/) มี ratio ผิด/ล้าสมัยถึง 7 ตัว (เช่น CRWD80: เก่า 4,000 จริง 250 — ถ้าใช้ค่าเก่าจะได้ Premium ผิดเป็น +1500% แทนที่จะเป็น +0.15% จริง) จึงยึดหลักไม่เชื่อไฟล์เก็บไว้แบบ static เด็ดขาด
  • ราคาหุ้นแม่และอัตราแลกเปลี่ยนมาจาก Yahoo Finance chart API (public, ไม่ใช้ API key) — อาจมี delay 15-20 นาทีสำหรับบางตลาด
  • ราคา DR จาก SET เป็นราคาปิดล่าสุด (`marketDateTime` ต่อแถว) ไม่ใช่ real-time ระหว่างเวลาทำการ
  • รายชื่อ DR ทั้งหมด (463 ตัว) มาจาก filter dropdown ของหน้า set.or.th/.../dr/marketdata เอง — ยืนยันแล้วว่าครบทุก issuer variant จริง (เช่น MSFT01/03/06/19/23/80 ทั้ง 6 ตัว, MICRON01/03/19/23/80 ทั้ง 5 ตัว) ไม่ใช่ list ที่กรองเฉพาะบาง issuer — resolve mapping หุ้นแม่ได้ครบทั้ง 463 ตัวแล้ว (2026-07-13): 334 → 385 จากคอลัมน์ "underlying" ในหน้า marketdata รายผู้ออก → 462 จากแผง "ชื่อเต็มสินทรัพย์อ้างอิง"/"ตลาดหลักทรัพย์"/"อัตราแปลงสภาพ" ในหน้า /dr/quote/<SYM>/price ของแต่ละ DR เอง → 463 (WORLDA01 ตัวสุดท้าย ผู้ใช้ช่วยหาหน้า Investing.com ของกองทุนมาให้ ยืนยันว่าคือ Invesco MSCI World UCITS ETF แล้วเจอ ticker MXWO บน London Stock Exchange ที่ใช้งานได้จริงบน Yahoo) — ทุกตัว cross-check สกุลเงินกับ exercisePriceUnit ของ SET เองก่อน ไม่เชื่อ currency ตาม exchange เฉยๆ (เจอ mismatch จริง 3 ครั้ง: BONDUS01, BONDAS19, WORLDA01 — ล้วนจดทะเบียนตลาดหนึ่งแต่ราคาสกุลเงินอื่น)
  • 6 ตัว (ADVANT19, ADVANT23, ADVANT80, NESTLE80, SYNP03, SYNP23) มี Yahoo ticker แต่ดึงราคาไม่สำเร็จระหว่าง scan — แสดงราคาจริงได้แต่ไม่มี Fair Price
  • ระหว่างเวลาทำการ SET หน้านี้ auto-refresh ทุก ~30 วิผ่าน GitHub Actions (scripts/dr_scan/scan_and_publish.py) — นอกเวลาทำการเป็น static snapshot ของรอบสุดท้ายก่อนตลาดปิด ต้อง trigger manual (`.tools/scan_dr.py` + `.tools/build_dr_page.py` ในโปรเจกต์เอง) ถ้าต้องการอัปเดตนอกเวลาทำการ
  • Premium/Discount ที่เห็นเป็นตัวเลขดิบ ยังไม่หักค่าธรรมเนียมซื้อขาย/สเปรด/ภาษี — ต้องคำนวณต้นทุนจริงเพิ่มก่อนตัดสินใจ arbitrage
  • Swing Stage / Target / Stop / Reward:Risk ไม่ใช่การนับคลื่น Elliott Wave ที่ยืนยันโดยนักวิเคราะห์ — คำนวณจาก pivot ราคาอัตโนมัติ (ZigZag แบบ threshold ปรับตามความผันผวน) + ฉายภาพ Fibonacci retracement/extension ล้วนๆ ไม่มีมนุษย์ตรวจสอบทีละตัว การนับคลื่นแบบ subjective จริงๆ แม้แต่นักวิเคราะห์มืออาชีพยังตีความไม่ตรงกันบนกราฟเดียวกัน — อัลกอริทึมนี้ตีความคลาดเคลื่อนได้เป็นเรื่องปกติ ไม่ใช่ edge case ใช้ประกอบการตัดสินใจเท่านั้น ไม่ใช่คำแนะนำซื้อขาย — คำนวณรายสัปดาห์ (ไม่ real-time เหมือนราคา) เก็บไว้ที่ `scripts/dr_scan/wave_cache.json` แสดงวันที่ข้อมูลล่าสุด (`ข้อมูล ณ ...`) ทุกครั้ง และ flag ทันทีถ้าราคาล่าสุดทะลุ target/stop ไปแล้วตั้งแต่คำนวณครั้งก่อน
